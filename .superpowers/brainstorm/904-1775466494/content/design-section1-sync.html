Section 1 — Data Transparency & Sync Status

Every page should answer: how fresh is this data, and did it load cleanly?

1a · Global Sync Status Bar

A persistent indicator in the sidebar or header shows the last successful sync time and a colour-coded health dot for each data source.

Sidebar — bottom section
Untapped.gg — synced 14 min ago
YGOProDeck — synced 2 hrs ago
MDM Tournaments — partial (3/5 events loaded)

1b · Per-Page Staleness Indicator

Each data-heavy page shows a small timestamp + freshness badge near the title. Goes amber if data is >2 hrs old, red if >12 hrs.

Dashboard — page header
Meta Dashboard ● Fresh — 14 min ago

1c · Partial-Failure Banner

When a sync completes but one source failed or returned incomplete data, a dismissible warning appears at the top of the page — not a full error, just a notice.

Page-level warning banner
⚠ MDM tournament data is incomplete — 3 of 5 recent events loaded. Some matchup stats may not reflect the full picture. Retry sync

Backend change needed: the sync service will track per-source status (success / partial / failed) and timestamps in the database, exposed via a new GET /api/sync/status endpoint the frontend polls on mount.
