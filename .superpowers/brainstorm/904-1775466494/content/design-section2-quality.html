Section 2 — Code Quality Foundations

Targeted fixes that make the app more reliable and easier to build on — not a full rewrite.

2a · Replace Silent Error Catches

Several components currently have .catch(() => {}) — errors disappear silently. Replace with proper error state that surfaces a message to the user via the existing ErrorBanner component.

Before → After
// Before
fetchDecks().catch(() => {})
// After
fetchDecks().catch((err) => setError(err.message))
// + <ErrorBanner> rendered when error state is set

2b · API Service Layer

Components currently call Axios directly. Extract calls into the existing api/ directory pattern (like api/cards.ts and api/meta.ts) so components stay thin and API shape changes only need fixing in one place.

Files to add: api/matchups.ts, api/sync.ts, api/tournaments.ts

2c · Request Cancellation

Add AbortController to useEffect data-fetch hooks so in-flight requests are cancelled when a component unmounts. Prevents state updates on unmounted components (a common React warning in this codebase).

2d · Targeted Tests for Critical Paths

No tests currently exist. Rather than broad coverage, focus on the highest-value areas:

  • Sync status logic (is data fresh? partial failure detection)
  • Matchup blending calculation (Untapped + tournament merge)
  • Fuzzy deck matching utility (already complex, already in the codebase)

Using Vitest — already compatible with the Vite setup, zero extra config needed.
